# Vibe Engineering

**Verify It Before Everything.**

Vibe Engineering is a sovereign AI building method for non-technical creators who want the speed of AI without surrendering quality, judgment, or ownership.

> Power without principles breeds chaos.  
> Freedom without ownership is just another dependency.

## Product promise

- Stop vibe coding. Start vibe engineering.
- Learn the few principles that matter.
- Lock those principles into an automated build system.
- Verify architecture, behavior, safety, usability, and taste before release.
- Give the customer complete ownership of code, data, accounts, and workflows.

## The eight-stage flow

1. Vision
2. Blueprint
3. Build
4. Verify
5. Council
6. Judge
7. Ship
8. Improve

## The Vibe system

- **Vibe Spec** — plain-language build contract
- **Vibe Flow** — automated quality gates
- **Vibe Council** — multi-perspective review
- **Vibe Judge** — final release decision
- **Vibe Score** — visible quality standard

## Vibe Score

Projects are scored for:

- Clarity
- Reliability
- Security
- Maintainability
- Taste
- Ownership

Recommended release floor: **8.5/10**.

## Run locally

```bash
npm install
npm run dev
```

## Production build

```bash
npm run build
```

The generated `dist/` directory can be deployed to Vercel or any static host.
