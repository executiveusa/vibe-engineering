# Coding standards

- Preserve existing architecture and conventions unless an approved issue authorizes change.
- Prefer the smallest understandable implementation.
- Reuse before adding dependencies or abstractions.
- Avoid unrelated refactors.
- Validate untrusted input and handle expected failures explicitly.
- Do not silently swallow errors.
- Add tests proportional to behavioral and operational risk.
- Keep secrets, credentials, personal data, and tokens out of code, logs, prompts, evidence, and commits.
- Treat accessibility, responsive behavior, ownership, portability, and rollback as release concerns.
- Run the repository's documented checks before committing.
- No evidence means no completion claim.
