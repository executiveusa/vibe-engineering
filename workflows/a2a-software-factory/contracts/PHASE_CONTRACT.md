# OPERATING CONTRACT — PHASE CONTRACT

A **Phase** is a coherent, independently verifiable increment of work.

## Mandatory Phase Attributes

1. `PHASE ID`: Unique identifier (e.g., `PHASE-01-AUTH`).
2. `OBJECTIVE`: Clear technical and functional goal.
3. `CONSEQUENCE LEVEL`: `LOW`, `MEDIUM`, or `HIGH` per Consequence Law.
4. `ALLOWED SCOPE`: Explicit list of directories/files allowed for modification.
5. `PROHIBITED SCOPE`: Explicit list of files/systems forbidden from modification.
6. `ACCEPTANCE CRITERIA`: Binary pass/fail conditions.
7. `HARNESS REQUIREMENTS`: Exact test suites and validation commands required.
8. `ROLLBACK STRATEGY`: Automated or manual steps to revert changes safely.
