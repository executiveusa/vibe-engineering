# Example Vibe Spec

## Product promise

Help a non-technical local organization publish a clear bilingual service site that it can operate and move without vendor permission.

## Owner

The organization controls the repository, hosting project, domain, analytics, content, and data exports.

## Primary user

A mobile visitor deciding whether the service is relevant and trustworthy.

## Primary journey

1. Understand the offer in five seconds.
2. Verify credibility and service area.
3. Select the correct action.
4. Submit only the minimum information required.
5. Receive a clear confirmation and next step.

## In scope

- responsive English and Spanish pages,
- accessible navigation,
- service explanation,
- contact conversion path,
- privacy-respecting analytics,
- agent-readable metadata,
- documented deployment and ownership handoff.

## Out of scope

- custom CRM,
- autonomous sales claims,
- payment processing,
- hidden lead resale.

## Acceptance evidence

- production build passes,
- core journeys work on mobile and desktop,
- keyboard navigation works,
- forms validate and show recovery states,
- Spanish pages contain no English leakage,
- repository and hosting access belong to the owner,
- rollback steps are documented.
