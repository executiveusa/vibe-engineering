# Grill-Driven Project Intake

This module turns repository onboarding into a governed sequence:

1. Capability check.
2. Read-only repository audit.
3. Project summary and confidence matrix.
4. One-question-at-a-time Grill interview.
5. Human-confirmed Vibe Project Contract.
6. Production-readiness PRD.
7. Optional Sandcastle dispatch for one approved ticket.

The agent must inspect available evidence before asking the founder for information. Questions are reserved for product intent, commercial decisions, ownership, risk tolerance, and unresolved contradictions.

## Format responsibilities

- Markdown: human-readable truth and policy.
- YAML: editable project configuration.
- JSON: validated state, audit output, and Judge results.
- POML: executable agent instructions.

## Hard stops

Do not dispatch implementation when:

- the repository purpose is unclear;
- the canonical repository is disputed;
- commercial or mission value is undefined;
- production readiness lacks a definition;
- a destructive or consequential decision requires owner approval;
- the audit evidence is incomplete.

## Grill behavior

Ask exactly one material question at a time. Each question must include:

- the unresolved decision;
- the agent's recommendation;
- the consequence of each major option;
- a direct request for the founder's decision.

Do not repeat questions already answered by repository evidence or prior founder decisions.
