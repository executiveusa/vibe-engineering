# Contributing

Contributions should strengthen clarity, verification, portability, accessibility, or ownership.

## Before changing code

1. Read the relevant documents in `docs/`.
2. Define the intended outcome and exact scope.
3. Preserve unrelated behavior.
4. Add or update evidence for changed release logic.

## Required checks

```bash
npm run check
```

## Pull request standard

Include:

- what changed,
- why it changed,
- user impact,
- verification evidence,
- known limitations,
- rollback notes,
- ownership or dependency impact.

Do not describe placeholders, mocks, or partial integrations as complete.
